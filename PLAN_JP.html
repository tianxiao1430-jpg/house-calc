House Calc
作成日: 2026年3月28日  |  ステータス: DRAFT  |  バージョン: 1.0

1. プロジェクト概要

在日中国人向けのスマートフォンアプリ。ユーザーが「賃貸」または「購入」モードを選択後、不動産サイトの物件情報をスクリーンショットで撮影すると、AIが自動的に情報を読み取り、対話形式で個人情報をヒアリングした上で、その物件の月額費用の完全な内訳を算出する。「購入 vs 賃貸」の比較ツールではなく、すでに決断したユーザーが具体的なコストを把握するためのツール。

コンセプト
「グループチャットの不動産に詳しい友人」を置き換える

2つのモード

モード計算内容
賃貸家賃 + 管理費 + 更新料按分 + 保険 + 保証会社 = 毎月の実質支出 + 初期費用総額
購入ローン返済 + 管理費 + 修繕積立金 + 固都税 + 保険 = 毎月の実質支出 + 初期費用総額

2. ターゲットユーザー

  • 日本在住の中国語話者（就労ビザ・永住権・留学生など）
  • 賃貸または購入を決めた上で、具体的な物件の実質月額コストを知りたい方
  • 日本の不動産費用構造に詳しくなく、中国語での説明を必要とする方

3. コア機能（MVP）

機能説明
モード選択「賃貸」または「購入」を選択し、対応する計算フローに進む
スクリーンショット認識ユーザーが物件画面を撮影/選択 → AIがモードに応じた情報を自動抽出
対話型ヒアリングAIが中国語でモード別の個人情報を質問（購入：頭金・ローン年数、賃貸：敷金・礼金等）
費用明細レポートその物件の月額支出完全内訳 + 初期費用 + 長期累計
履歴管理直近20件の計算結果をローカル保存

4. 技術アーキテクチャ

主要な設計判断

  • 費用計算は決定的なTypeScriptコードで実行（LLM非依存 → 正確性 + コスト抑制）
  • LLMの役割：スクリーンショット認識 + 自然言語対話のみ
  • バックエンドサーバー不要、API直接呼び出し（MVP段階）
  • 金利デフォルト 0.475%（変動金利）、固定資産税評価額 = 市場価格 × 70%

6. 費用計算モデル詳細（コアIP）

購入側：月額支出

月額返済額  = 元利均等返済（借入額, 金利0.475%, 返済年数）
管理費      = スクショから抽出
修繕積立金  = スクショから抽出（固定値）
固定資産税/月 = (市場価格 × 0.70 × 0.014) / 12
都市計画税/月 = (市場価格 × 0.70 × 0.003) / 12
火災保険/月  = 専有面積 × 350円 / 12（概算）
団信       = 金利に含む（変動金利は通常含む）

購入側：初期費用

仲介手数料      = 物件価格 × 3% + 6万円 + 消費税10%
登録免許税      = 評価額 × 0.3%（住宅軽減）
不動産取得税    = 評価額 × 3%（住宅軽減後）
印紙税         = 1万〜6万円（価格帯別）
司法書士報酬    = 10万〜15万円（概算）
ローン事務手数料 = 借入額 × 2.2%

賃貸側：月額支出

家賃          = ユーザー入力
管理費/共益費   = 物件情報に含まれる場合あり、デフォルト0
火災保険/月    = 約1.5万円/年 / 12 = ¥1,250
更新料/月按分  = 家賃 × 1 / 24（2年に1回）
保証会社/月    = 家賃 × 0.5 / 12（年次更新50%）

賃貸側：初期費用

敷金          = 家賃 × 1〜2ヶ月（デフォルト1）
礼金          = 家賃 × 0〜1ヶ月（デフォルト1）
仲介手数料     = 家賃 × 1.1（税込）
火災保険      = 約1.5万円/年
保証会社初回   = 家賃 × 50%
鍵交換        = 約1.5万〜2万円

8. リスクと対策

リスク確率影響度対策
AIスクショ認識の精度不足 中 高 手動入力フォールバック、プロンプト最適化、対応プラットフォーム限定
APIコスト超過 低 中 費用計算はローカルコード、Vision結果キャッシュ、無料利用回数制限
App Store審査却下 中 中 審査ガイドラインの事前調査、プライバシーポリシー完備
費用モデルの計算不正確 低 高 実際の不動産仲介の見積もりと検証、「参考値」表示
ユーザーが質問に回答しない 低 中 合理的なデフォルト値を設定、最大5ターンで結果出力
